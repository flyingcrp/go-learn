# AGENTS.md

## 工作方式

- 先读最小相关上下文。
- 优先沿用仓库已有模式，不要乱加抽象。
- 改动保持小而集中。

## 代码规则

- `handler` 只做 HTTP 收发。
- `service` 只放业务逻辑。
- `repo` 只做数据访问。
- 能复用就复用 `internal/common/*`。

## 完成前

- 执行 `gofmt -w cmd internal`。
- 执行 `go test ./...`。
- 不要留下已知问题。

## 安全边界

- 不要重写无关文件。
- 非必要不要新增依赖。
- 如果改动会影响行为，除非任务要求，否则保持原有契约。
